## 1. 디지털 정보의 표현


### 🧮 디지털 정보의 개념
- 디지털(digital): 수치로 표현
- 아날로그(analog): 연속적인 값으로 표현
- 디지털 정보는 **이진수(0과 1)**로 구성됨

### 💡 디지털 정보의 표현 방식
- 켜진 상태 (1): 참(True)
- 꺼진 상태 (0): 거짓(False)

### 📏 정보의 표현 단위
- 비트(bit; binary digit): 가장 작은 정보 단위
- 0과 1 중 하나의 상태
- 바이트(byte): 8개의 비트로 구성

### 🔢 이진수(binary)와 십진수(decimal)
- 십진수(365): 3×100 + 6×10 + 5×1
- 이진수: 1, 0의 조합으로 숫자 표현

### 🧩 16진수(Hexadecimal)
- 0~9, A(10)~F(15)로 구성
- 4비트로 1자리 표현 (예: 0x9B = 155)

### 🔡 문자 표현 (ASCII, Unicode)
- ASCII: 영문, 숫자, 특수기호 표현
- Unicode: 한글 및 다양한 언어 문자 표현

### 📊 데이터 크기 단위
- K (kilo): 1,024 (2¹⁰)
- M (mega): 1,048,576 (2²⁰)
- G (giga): 1,073,741,824 (2³⁰)

## 🔄 보수(Complement) 개념

### 1. 보수란?
- 보수(complement)는 쉽게 말해 숫자를 뒤집거나 변환해서 더 쉽게 계산할 수 있게 만드는 도구
- 특히, 뺄셈을 덧셈으로 바꾸는 데 많이 사용

### 2. 왜 보수를 사용할까?
- 컴퓨터는 덧셈은 쉽게 잘하지만, 뺄셈은 상대적으로 복잡.
- 그래서 뺄셈을 덧셈으로 바꿔서 처리하는 방식 등장.
- 이게 바로 보수 개념.

### 3. 1의 보수 (One's Complement)
- 정의
    - 1의 보수는 숫자의 모든 비트를 반전(0은 1로, 1은 0으로) 시키는 방식
    - 모든 비트를 반전 (0 ↔ 1)
    - 예: 0101 → 1010
- 이유
    - 이렇게 뒤집으면 더했을 때 1의 자리에서 올림이 발생해서 결과적으로 뺄셈과 같은 효과

- 예시

    ```
    5 (2진수)  =  0101
    1의 보수  =  1010
    ```

    - 0은 1로, 1은 0으로 변환


### 4. 2의 보수 (Two's Complement)
- 정의
    - 2의 보수는 1의 보수에 1을 더하는 방식
- 이유
    - 이렇게 하면 음수를 표현하고, 덧셈으로 뺄셈을 더 정확하게 처리
- 예시
    ```
    5 (2진수)   =  0101
    1의 보수   =  1010
    2의 보수   =  1010 + 1 = 1011
    ```
    - 1011은 -5를 의미

### 5. 비교: 1의 보수 vs 2의 보수
- 1의 보수는 단순히 반전만 시키는 방식.
- 2의 보수는 반전을 먼저 하고, 반전된 숫자에 1을 더하는 방식.
- 2의 보수가 더 많이 사용
    - 왜냐하면 올림 처리가 자동으로 이루어져서 결과가 정확하게 나오기 때문

# 컴퓨터 연산 하드웨어

## 1. 디지털 정보
- 디지털은 1과 0으로 표현
- 켜짐(1) 과 꺼짐(0) 상태로 구분
- 전구가 켜지면 1, 꺼지면 0

## 2. 이진수와 비트
- 컴퓨터는 모든 정보를 비트(bit)로 표현
- 비트는 binary digit의 줄임말
- 이진수는 0과 1만 사용
    - 예를 들어 1010은 이진수
- 바이트(byte)는 8개의 비트로 구성
    - 11001010 같은 형태입니다.

## 3. 논리 연산 (AND, OR, NOT)
- 논리 연산은 명제(참/거짓)를 다루는 방식
    - 예를 들어:
        - AND: 둘 다 참이어야 참
        - OR: 둘 중 하나라도 참이면 참
        - NOT: 참이면 거짓, 거짓이면 참
- 예시로, 비가 오는 날 AND 우산을 가져야 젖지 않음
    - 비 AND 우산 = 안 젖음
    - 비 AND 우산 없음 = 젖음

## 4. 논리 게이트
- 논리 연산을 실제 회로로 구현하는 것이 논리 게이트
- AND 게이트는 두 입력이 모두 1이어야 출력이 1
- OR 게이트는 입력 중 하나라도 1이면 출력이 1
- NOT 게이트는 입력을 반전시켜 0이면 1, 1이면 0으로 바꿉니다.

## 5. 반가산기 (Half Adder)
- 덧셈을 수행하는 회로
- 두 개의 비트를 더해 합(sum)과 자리 올림(carry)을 계산
- 예:
    - 1 + 1 = 10 → 합은 0, 자리 올림은 1

## 6. 가산기와 감산기
- 가산기는 덧셈, 감산기는 뺄셈을 수행
- 감산기는 가산기와 비슷하지만, 빼기를 수행할 때 보수를 사용

## 7. ALU (산술 논리 장치)
- ALU(Arithmetic Logic Unit)는 CPU의 핵심 부분으로, 산술 및 논리 연산을 수행
- 덧셈, 뺄셈, 곱셈, 나눗셈과 논리 연산을 담당

## 5. 튜링 기계와 폰 노이만 컴퓨터

### 📌 컴퓨터의 발명
- 컴퓨터는 한 사람이 발명한 게 아니라 다양한 기술 발전을 통해 탄생
- 결정적 역할을 한 두 인물:
- 앨런 튜링: 계산 원리를 정립, 튜링 기계(Turing Machine) 제안
- 존 폰 노이만: 현재 컴퓨터 구조(폰 노이만 아키텍처)의 기반 설계

### 📌 튜링 기계 (Turing Machine)
- 모든 컴퓨터 알고리즘을 실행할 수 있는 이론적 모델
- 테이프에 데이터를 읽고, 쓰고, 지우면서 계산 수행
- 현재 컴퓨터 작동 원리의 기초

### 📌 폰 노이만 컴퓨터
- 프로그램 저장 방식 컴퓨터 구조
- 프로그램과 데이터를 메모리에 저장하고 CPU가 이를 순차적으로 실행
- 현재 대부분의 컴퓨터가 이 구조를 따름
